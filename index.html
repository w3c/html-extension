These applicable extension specifications change conformance for HTML. They introduce or change the processing requirements of elements, attributes, attribute values, or interface APIs.

This document is proposed as the basis for a Working Group Note. It is maintained on Github. If you find a problem or want to make a suggestion, please file an issue.

Introduction

The specifications listed here extend the HTML specification.

W3C Recommendations that extend HTML

CSS Style Attributes

Markup languages such as HTML and SVG provide a style attribute on most elements, to hold inline style information that applies to those elements. This draft describes the syntax and interpretation of the CSS fragment that can be used in such style attributes.

Image Description (longdesc)

This specification defines a longdesc attribute (based on the longdesc attribute of HTML 4) to link descriptions to images in HTML content.

Pointer Events

This specification defines a unified event model to handle mouse, touch and stylus input. It builds on mouse events, and extends the Element, GlobalEventHandlers and Navigator interfaces of HTML.

RDFa

RDFa comprises a family of specifications which specify attributes for XML, SGML, and HTML-style markup languages, and a processing model that can reliably parse RDFa information to produce RDF

Subresource Integrity

This specification defines a mechanism by which user agents may verify that a fetched resource has been delivered without unexpected manipulation. It adds the integrity attribute for link elements whose rel attribute value inclues stylsheet, and for script elements, and extends the HTMLLinkElement and HTMLScriptElement interfaces. [[SRI]]

Webmention

This specification defines a mechanism to notify a URL that it has been mentioned. It adds the webmention value for the rel attribute of links.

Content Security Policy Level 3

This document defines a mechanism by to control the resources a particular page can fetch or execute, as well as a number of security-relevant policy decisions. It introduces an HTTP header Content-Security-Policy which can be used as a value for the meta element's http-equiv attribute, along with parameters to be used in the corresponding content attribute, that effect navigation and the use of resources external to the page.

HTML Media Capture

The HTML Media Capture specification defines an HTML form extension that facilitates user access to a device's media capture mechanism, such as a camera, or microphone, from within a file upload control.

Microdata

This specification defines several global attributes for HTML, to add machine-parseable semantics to HTML content, in a similar way to RDFa but with less power and expressivity.

Preload

This specification defines a preload keyword as a value of the rel attribute for links. The keyword provides a declarative fetch primitive that initiates an early fetch and separates fetching from resource execution.

Presentation API

This specification defines an API to enable web content to access external presentation-type displays and use them for presenting web content. It extends the Navigator interface.

Public Identifiers for Entity Resolution in XHTML

Merging the content of this specification directly into HTML is now tracked as HTML Issue #259.

This document adds the additional public identifier -//W3C//ENTITIES HTML MathML Set//EN//XML for XHTML user agents, and causes the HTML character entity definitions to be loaded by explicitly referencing the set of definitions that is used by HTML.

Referrer Policy

This document describes how an author can set a referrer policy for documents they create, and the impact of such a policy on the Referer HTTP header for outgoing requests and navigations.

Resource Hints

This specification defines the dns-prefetch, preconnect, prefetch, and prerender values of the rel attribute for links (<link>). These primitives enable the developer, and the server generating or delivering the resources, to assist the user agent decide which origins it should connect to, and which resources it should fetch and preprocess to improve page performance.

Other documents that extend HTML

HTTP Client Hints

This document defines a set of new request header fields that allow the client to perform proactive content negotiation (Section 3.4.1 of [RFC7231]) by indicating a list of device and agent specific preferences, through a mechanism similar to the Accept header field which is used to indicate preferred response formats.
